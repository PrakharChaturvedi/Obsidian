## Symmetric key cryptography
- The same key is used for both encryption and decryption of data.
- This means that the sender and receiver must share the same secret key in order to communicate securely.
- Examples of symmetric key algorithms include *DES (Data Encryption Standard) and AES (Advanced Encryption Standard).*

## Public key cryptography
- Public key cryptosystems use a pair of mathematically related keys: a public key for encryption and a private key for decryption. 
- The public key can be freely distributed, while the private key must be kept secret.
- Any message encrypted with the public key can only be decrypted using the corresponding private key. 
- Examples of public-key algorithms are *RSA (Rivest-Shamir-Adleman) and ECC (Elliptic Curve Cryptography).*

## Hash Functions Description 
- <font color="#00b050">Criteria of good hash functions :</font>
	- <font color="#ffc000">Preimage resistance</font>
		- Preimage resistance is **the property of a hash function that it is hard to invert**, that is, given an element in the range of a hash function, it should be computationally infeasible to find an input that maps to that element.
	- <font color="#ffc000">Secondary preimage resistance</font>
		- The property of a hash function that it is computationally infeasible to find any second input that has the same output as a given input.
	- <font color="#ffc000">Collision resistance</font>
		- A collision occurs when more than one value to be hashed by a particular hash function hash to the same slot in the table or data structure (hash table) being generated by the hash function. 
		- Collision resistance is **the property of a hash function that it is computationally infeasible to find two colliding inputs**. 
- <font color="#00b050">Avalanche Effect :</font>
	- The avalanche effect refers to a phenomenon in cryptography where a small change in the input of a cryptographic algorithm produces a significantly different output.
	- This property is desirable as it adds security by making it difficult for an attacker to deduce any information about the original input based on the encrypted output.
- Important hashing algorithms 
	- MD4 and MD5
		- MD5 :
			- It is *a cryptographic protocol used for authenticating messages as well as content verification and digital signatures*. 
			- MD5 is based on a hash function that verifies that a file you sent matches the file received by the person you sent it to.
			- Fast and produces 128 bits message digests.
			- <font color="#92d050">Example :</font> “An indie developer publishes a game free of charge. The game file has a specific hash value assigned. You’re downloading the game from a third-party site. If the hash of the downloaded file differs, it isn’t the original one. Thus, it can be a virus, or files may have been damaged while downloading (e.g., due to network issues).”
			- <font color="#92d050">Working of MD5 :</font> 
				- Padding
					- Original message + (Padding) [Padding => extra bits]
					- So the total length becomes 64 bits less than exact multiple of 512.
					- Example : Message  size = 1000 bits 
						- then 512-64 = 448 (less than 1000)
						- 1024-64 = 960 (less than 1000) 
						- 1536 - 64 = 1472 So, 1472-1000= 472 these bits needs to be added  
				- Appending 
					- Append the original length before padding
					- Calculate length mod 64 [In most cases we get 64 bits as answer]
						- Example : 1000 mod 64 
					- So it again becomes 512 bits
				- Dividing [Each 512 bits] 
					- After performing 2nd step we get multiple block of multiple 512 length.
					- hence, blocks of length = 512 blocks is divided.
				- Initializing [4 chaining variables]
					- each chaining variable is of 32 bit length
					- with value of each variable [A, B, C, D] being predefined
				- Processing [512 blocks]
					- Copy 4 chaining variables into some corresponding variables.
					- A = a, B = b, C = c and D = d
					- Divide 512 bit blocks into 16 blocks each of size 23 bits.
				- 4 rounds 
					- 16 sub-Blocks and a constant (k)
					- 16 Sub-Blocks-> One round <- Constant(K)

									  a  b c d
					- a = b+(a + process,P(b,c,d)+m[i]+T[k])

				- The MD5 is no longer recommended for solutions requiring a high level of security.
				- The MD5 algorithm has security vulnerabilities, and it’s considered cryptographically broken.
				- Nowadays, there are more secure algorithms like SHA-2.
	- SHA : 
		- The Secure Hash Algorithm with a digest size of 256 bits, or the SHA 256 algorithm, is one of the most widely used hash algorithms.
		- SHA-512 is the version of SHA with a 512-bit message digest. While there are other variants.

			- SHA-256 meets all of the important requirements of the cryptographic hash function.
			- Algorithm :
				1) *Padding Bits:* It adds some extra bits to the message, such that the length is exactly 64 bits short of a multiple of 512.
				2) *Padding Length:* You can add 64 bits of data now to make the final plaintext a multiple of 512
				3) *Initializing the Buffers:* You need to initialize the default values for eight buffers (a-h, 32 bits each) to be used in the round.
				4) *Initializing the Keys:* You also need to store 64 different keys in an array, ranging from K[0] to K[63].
				5) *Process the blocks:* 
					1) The entire message gets broken down into multiple blocks of 512 bits each.
					2) It puts each block through 64 rounds of operation, with the output of each block serving as the input for the following block.

## *Key-generation and Distributions
- Key management is the process of managing cryptographic keys in a cryptosystem. 
- It includes:
	- Cryptographic protocol design
	- Key servers
	- User procedures
- Distribution of public keys :
	- <font color="#ffc000">Public announcement</font>
		- Users distribute public keys to recipients or broadcast to community at large.
		- Major weakness is forgery  
			- Anyone can create a key claiming to be someone else and broadcast it. Until forgery is discovered can masquerade as claimed user.
	- <font color="#ffc000">Publicly available directory</font>
		- **The key distribution of public keys is done through public key servers**. 
		- When a person creates a key-pair, they keep one key private and the other, known as the public-key, is uploaded to a server where it can be accessed by anyone to send the user a private, encrypted, message.
	- <font color="#ffc000">Public-Key Authority</font>
		-  It is similar to the directory but, improves security by tightening control over the distribution of keys from the directory. It requires users to know the public key for the directory. 
		- Whenever the keys are needed, real-time access to the directory is made by the user to obtain any desired public key securely.
	- <font color="#ffc000">Public-key certificates</font>
		- This time authority provides a certificate to allow key exchange without real-time access to the public authority each time.
		- The certificate is accompanied by some other info such as period of validity, rights of use, etc. All of this content is signed by the private key of the certificate authority and it can be verified by anyone possessing the authority’s public key.
## Diffie-Hellman Key Exchange
- The Diffie-Hellman key exchange was the **first widely used method of safely developing and exchanging keys over an insecure channel**.![[Pasted image 20231201193748.png]]
- A public-key distribution scheme cannot be used to exchange an arbitrary message rather it can establish a common key known only to the two participants.
- Example :
	- Users Alice & Bob who wish to swap keys.
	- Agree on prime q=353 and α=3
	- Select random secret keys: 
		- A chooses xA=97 
		- B chooses xB=233
	- Compute public keys:
		- yA =3 97 mod 353 = 40 (Alice) 
		- yB =3 233 mod 353 = 248 (Bob)
	- Compute shared session key as:
		- KAB= yB xA mod 353 = 248^(97) = 160 (Alice)
		- KAB= yA xB mod 353 = 40^(233) = 160 (Bob)

## X.509 Authentication Services
- PKIX stands for Public Key Infrastructure X.509 .It is a working group under the Internet Engineering Task Force (IETF). The group's goal is to support public key infrastructures based on X.509 on the Internet.![[Pasted image 20231201202841.png]]
- **A certificate-based authentication security framework that can be used for providing secure transaction processing and private information**. These are primarily used for handling the security and identity in computer networking and internet-based communications.![[Pasted image 20231201195100.png]]


X.509 is a standard that defines the format of public-key certificates. These certificates are used for various security purposes, such as secure email, web browsing, and network communications. The X.509 standard is widely used in the implementation of public key infrastructure (PKI).

Here are some key points about X.509 certificates:

1. **Structure:** X.509 certificates have a hierarchical structure and consist of several fields, including:

   - **Version:** Indicates the version of the X.509 standard used.
   - **Serial Number:** A unique identifier for the certificate issued by the certificate authority (CA).
   - **Signature Algorithm:** The algorithm used by the CA to sign the certificate.
   - **Issuer:** Identifies the entity that issued the certificate.
   - **Validity Period:** Specifies the time during which the certificate is valid.
   - **Subject:** Identifies the entity the certificate is issued to.
   - **Public Key Information:** Includes the public key and the algorithm associated with it.
   - **Issuer Signature:** The digital signature created by the CA to validate the certificate.

2. **Public Key Infrastructure (PKI):** X.509 certificates are a fundamental component of PKI, which is a system for managing digital keys and certificates. PKI enables secure communication by using cryptographic keys and digital certificates.

3. **Certificate Authorities (CAs):** CAs are trusted entities that issue and verify digital certificates. The CA's digital signature on the certificate attests to the authenticity of the information contained in the certificate.

4. **Certificate Revocation:** X.509 certificates can be revoked before their expiration date if the private key associated with the certificate is compromised or for other reasons. Certificate Revocation Lists (CRLs) or Online Certificate Status Protocol (OCSP) are mechanisms used to check the revocation status of certificates.

5. **Usage:** X.509 certificates are used in various security protocols and applications, including:

   - **SSL/TLS:** Used to secure web communication.
   - **S/MIME:** Used for securing email communication.
   - **IPsec:** Used for securing network communication.
   - **Code Signing:** Used to verify the authenticity of software and code.
   - **Authentication:** Used for user and device authentication.

6. **File Formats:** X.509 certificates are often stored in different file formats, such as PEM (Privacy Enhanced Mail), DER (Distinguished Encoding Rules), and PFX/PKCS#12.

Understanding X.509 certificates is crucial for anyone involved in implementing or managing secure communication systems.
